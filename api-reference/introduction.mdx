## Vue d'ensemble

L'API VoiceHire vous permet d'automatiser vos campagnes de recrutement vocal directement depuis vos outils existants. Créez des campagnes, ajoutez des candidats, et recevez les résultats d'entretiens en temps réel.

## Démarrage rapide

### 1. Obtenir votre clé API

Connectez-vous à votre [tableau de bord VoiceHire](https://app.voicehire.io) et accédez à la section API pour générer votre clé.

### 2. Authentification

Incluez votre clé API dans l'en-tête de chaque requête:

```bash
X-API-Key: vh_live_XXXXXXXXXXXXX
```

### 3. Première requête

Testez votre intégration en récupérant vos crédits disponibles:

```bash
curl -X GET https://app.voicehire.io/api/v1/account/credits \
  -H "X-API-Key: vh_live_XXXXXXXXXXXXX"
```

## Flux de travail typique

1. **Créer une campagne** - Définissez le poste et les questions d'entretien
2. **Ajouter des candidats** - Importez vos candidats (jusqu'à 100 par requête)
3. **Suivre le statut** - Consultez l'avancement de votre campagne
4. **Récupérer les résultats** - Obtenez les synthèses d'entretien et scores

## Endpoints disponibles

L'API est organisée en trois sections principales :

### Campagnes

Gérez vos campagnes de recrutement :
- Créer des campagnes avec génération automatique de questions
- Lister et filtrer vos campagnes
- Suivre le statut et les statistiques
- Modifier le statut (active, en pause, terminée)

### Candidats

Gérez les candidats de vos campagnes :
- Ajouter des candidats (déclenche automatiquement les appels)
- Sourcer des candidats par SMS
- Récupérer la liste des candidats avec filtres
- Accéder aux détails complets incluant les transcriptions

### Compte

Gérez votre compte et vos crédits :
- Consulter vos crédits disponibles
- Voir votre type d'abonnement
- Suivre votre consommation

## Format des requêtes

### Headers requis

```http
X-API-Key: vh_live_VOTRE_CLE_API
Content-Type: application/json
```

### Corps de requête

Les requêtes POST et PATCH doivent envoyer un JSON valide :

```json
{
  "job_title": "Commercial B2B",
  "contract_type": "CDI",
  "job_description": "Description du poste..."
}
```

## Format des réponses

### Réponse réussie

```json
{
  "campaign_id": "CAMP-20250701-ABC123",
  "status": "active",
  "candidate_link": "https://app.voicehire.io/postuler/CAMP-20250701-ABC123"
}
```

### Réponse d'erreur

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Description de l'erreur",
    "details": {
      "field": "Information supplémentaire"
    }
  }
}
```

## Codes de statut HTTP

| Code | Description |
|------|-------------|
| 200 | Requête réussie |
| 201 | Ressource créée avec succès |
| 400 | Requête invalide |
| 401 | Non authentifié |
| 403 | Accès refusé |
| 404 | Ressource non trouvée |
| 429 | Trop de requêtes |
| 500 | Erreur serveur |

## Pagination

Les endpoints qui retournent des listes supportent la pagination :

```
GET /api/v1/campaigns?page=2&per_page=50
```

- `page` : Numéro de page (défaut: 1)
- `per_page` : Éléments par page (défaut: 20, max: 100)

## Limites de taux

- 60 requêtes par minute
- 1000 requêtes par heure

En cas de dépassement, vous recevrez une erreur 429 avec un header `Retry-After`.
